# CI Qualite Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Mettre en place la CI GitHub Actions complete de real-fiscal : PHPUnit (+ smoke test), php-cs-fixer, twig-cs-fixer, PHPStan niveau 8, eslint + prettier, avec les memes verifications executables en local via scripts composer/npm.

**Architecture:** Un seul workflow `.github/workflows/ci.yml` a 3 jobs paralleles (`php-lint`, `php-tests`, `js-lint`) qui appellent exactement les scripts composer/npm definis pour le local. Outils PHP en require-dev, outils JS en devDependencies d'un package.json minimal.

**Tech Stack:** Symfony 8.1 / PHP 8.4, PHPUnit 13, friendsofphp/php-cs-fixer, vincentlanglet/twig-cs-fixer, phpstan/phpstan (+ phpstan-symfony), eslint 10 (flat config) + prettier 3, GitHub Actions.

## Global Constraints

- Spec source : `docs/superpowers/specs/2026-07-06-ci-setup-design.md` — toute divergence doit y etre reportee.
- Branche de travail : `chore/setup-ci` (deja creee, contient la spec). Tous les commits vont sur cette branche.
- Messages de commit : anglais, imperatif, format `type: description` (ex. `chore: add phpstan config`).
- PHP 8.4 / Symfony 8.1 (Flex contraint les paquets symfony/* a `8.1.*` via `extra.symfony.require`).
- En CI : linters en mode check-only, jamais d'autofix. Pas de base de donnees.
- `composer.lock` et `package-lock.json` sont commites a chaque changement de dependances.
- Ne PAS commiter : `CLAUDE.md`, `.claude/` (fichiers untracked hors perimetre), `node_modules/`, caches d'outils.
- PHPStan niveau 8. php-cs-fixer ruleset `@Symfony` (non-risky). Prettier `singleQuote: true` (aligne sur le style des fichiers recette Symfony).

### Task 3: php-cs-fixer

**Files:**
- Create: `.php-cs-fixer.dist.php`
- Modify: `composer.json` (require-dev + scripts), `composer.lock`, `.gitignore`

**Interfaces:**
- Consumes: rien
- Produces: scripts `composer cs` (check, exit != 0 si violation) et `composer cs:fix` (autofix) — `composer cs` est utilise par le job `php-lint` en Task 7

- [ ] **Step 1: Installer php-cs-fixer**

Run: `composer require --dev friendsofphp/php-cs-fixer`
Expected: installation sans erreur, `composer.lock` mis a jour.

- [ ] **Step 2: Creer la config**

Creer `.php-cs-fixer.dist.php` :

```php
<?php

$finder = (new PhpCsFixer\Finder())
    ->in([__DIR__.'/src', __DIR__.'/tests'])
;

return (new PhpCsFixer\Config())
    ->setRules([
        '@Symfony' => true,
    ])
    ->setFinder($finder)
;
```

Note : `migrations/` n'est pas dans le Finder (exclu du style par decision de spec).

- [ ] **Step 3: Ajouter les scripts composer**

Dans `composer.json`, section `scripts`, ajouter apres `"test"` :

```json
"cs": "php-cs-fixer fix --dry-run --diff",
"cs:fix": "php-cs-fixer fix"
```

(Composer prepend `vendor/bin` au PATH des scripts : pas de prefixe necessaire.)

- [ ] **Step 4: Verifier le passage sur le code existant**

Run: `composer cs`
Expected: exit 0. Si `src/Kernel.php` ou `tests/SmokeTest.php` sont flagges : lancer `composer cs:fix`, inspecter `git diff`, et inclure la correction dans le commit.

- [ ] **Step 5: Verifier la detection (violation volontaire)**

```bash
printf '<?php\nclass   BadStyle{function  x( ) {return 1 ;}}\n' > src/BadStyle.php
composer cs; echo "exit=$?"
rm src/BadStyle.php
```

Expected: diff affiche sur `src/BadStyle.php` et `exit=8` (non-zero). Le fichier est supprime apres verification.

- [ ] **Step 6: Ignorer le cache**

Ajouter a la fin de `.gitignore` :

```
###> friendsofphp/php-cs-fixer ###
/.php-cs-fixer.php
/.php-cs-fixer.cache
###< friendsofphp/php-cs-fixer ###
```

- [ ] **Step 7: Commit**

```bash
git add .php-cs-fixer.dist.php composer.json composer.lock .gitignore
git status --short   # verifier qu'aucun fichier non voulu (src/BadStyle.php, caches) n'est stage
git commit -m "chore: add php-cs-fixer with Symfony ruleset"
```

### Task 6: Toolchain JS (eslint + prettier)

**Files:**
- Create: `package.json`, `package-lock.json`, `eslint.config.mjs`, `.prettierrc`, `.prettierignore`
- Modify: `.gitignore`, potentiellement `assets/*.js` et `assets/controllers/*.js` (normalisation prettier)

**Interfaces:**
- Consumes: rien
- Produces: scripts `npm run lint`, `npm run lint:fix`, `npm run format`, `npm run format:check` — `lint` et `format:check` sont utilises par le job `js-lint` en Task 7

- [ ] **Step 1: Creer package.json**

Creer `package.json` :

```json
{
    "private": true,
    "scripts": {
        "lint": "eslint .",
        "lint:fix": "eslint . --fix",
        "format": "prettier --write \"assets/**/*.{js,css}\"",
        "format:check": "prettier --check \"assets/**/*.{js,css}\""
    }
}
```

Note : prettier est volontairement scope a `assets/` (et non `.`) pour ne pas entrer en conflit avec les YAML/JSON generes par Symfony Flex — raffinement de la cible enoncee dans la spec (`assets/**/*.js`), etendu au CSS du meme dossier.

- [ ] **Step 2: Installer les devDependencies**

Run: `npm install --save-dev eslint @eslint/js eslint-config-prettier globals prettier`
Expected: `package-lock.json` cree, `node_modules/` cree, 0 vulnerabilites bloquantes.

- [ ] **Step 3: Creer la config ESLint (flat config)**

Creer `eslint.config.mjs` :

```js
import js from '@eslint/js';
import prettier from 'eslint-config-prettier';
import globals from 'globals';

export default [
    {
        ignores: [
            'assets/vendor/**',
            'node_modules/**',
            'public/**',
            'var/**',
            'vendor/**',
            'graphify-out/**',
        ],
    },
    {
        files: ['assets/**/*.js'],
        languageOptions: {
            ecmaVersion: 'latest',
            sourceType: 'module',
            globals: { ...globals.browser },
        },
        rules: {
            ...js.configs.recommended.rules,
        },
    },
    prettier,
];
```

### Task 7: Workflow GitHub Actions

**Files:**
- Create: `.github/workflows/ci.yml`

**Interfaces:**
- Consumes: `composer cs` (Task 3), `composer twig:cs` (Task 4), `composer stan` (Task 5), `composer test` (Task 2), `npm run lint` + `npm run format:check` (Task 6)
- Produces: 3 checks GitHub (`php-lint`, `php-tests`, `js-lint`) sur PR et push master

- [ ] **Step 1: Creer le workflow**

Creer `.github/workflows/ci.yml` :

```yaml
name: CI

on:
    push:
        branches: [master]
    pull_request: ~

concurrency:
    group: ${{ github.workflow }}-${{ github.ref }}
    cancel-in-progress: true

jobs:
    php-lint:
        runs-on: ubuntu-latest
        steps:
            - uses: actions/checkout@v4
            - uses: shivammathur/setup-php@v2
              with:
                  php-version: '8.4'
                  coverage: none
            - uses: ramsey/composer-install@v3
            - run: composer cs
            - run: composer twig:cs
            - run: composer stan

    php-tests:
        runs-on: ubuntu-latest
        steps:
            - uses: actions/checkout@v4
            - uses: shivammathur/setup-php@v2
              with:
                  php-version: '8.4'
                  coverage: none
            - uses: ramsey/composer-install@v3
            - run: composer test

    js-lint:
        runs-on: ubuntu-latest
        steps:
            - uses: actions/checkout@v4
            - uses: actions/setup-node@v4
              with:
                  node-version: 22
                  cache: npm
            - run: npm ci
            - run: npm run lint
            - run: npm run format:check
```

(`ramsey/composer-install@v3` gere l'installation ET le cache composer.)

- [ ] **Step 2: Valider la syntaxe YAML localement**

Run: `python3 -c "import yaml, sys; yaml.safe_load(open('.github/workflows/ci.yml')); print('YAML OK')"`
Expected: `YAML OK`.

- [ ] **Step 3: Rejouer localement les commandes exactes des 3 jobs**

```bash
composer cs && composer twig:cs && composer stan   # job php-lint
composer test                                       # job php-tests
npm run lint && npm run format:check                # job js-lint
```

Expected: tout passe (exit 0). C'est la garantie que la CI sera verte du premier coup.

- [ ] **Step 4: Commit**

```bash
git add .github/workflows/ci.yml
git commit -m "ci: add github actions workflow with php-lint, php-tests and js-lint jobs"
```
